## 2. v0.5.1 关键变化（相对 v0.5.0）

### 2.1 实录：B精简版·角色轻标注 + 抗空返回
- **背景**：v0.5.0 按「说话人轮次」切分、并信任原稿的机器发言人编号。实测发现 ASR 的发言人识别**常不准**（同一个人被打成不同编号），导致同一人的连续发言被切得很碎。
- **新做法（B精简版）**：**不再相信原稿发言人编号**——切分改回按句子长度分块（`split_by_length`），由模型**按语义**判断说话人：
  - 同一人连续发言**合并成连贯段落**；课堂讲授大段直接出连贯正文、不加标签；
  - 仅在**确有切换处**（提问/问答互动）用**角色**加粗标注（`**讲师：**`/`**提问：**`/`**回答：**`/`**主持人：**`），角色来自语义、天然跨块一致，避开"全局发言人编号一致性"这个难点。
- **修复"部分段落残留发言人/时间戳"**：根因是部分分块**模型返回空内容**（中转/模型高并发回空 body，非异常），旧逻辑 `txt or 原文` 把**原始块**直接灌入。三道防线：
  1. `_correct_chunk`：空返回/报错**重试**（每块至多 `CORRECT_RETRIES=3` 次）；
  2. 送模型前 `strip_timestamps` **确定性去时间戳**（保留发言人标签作弱提示）——无论模型是否配合，时间戳都不会残留；
  3. 仍失败则 `clean_fallback`（去标签+时间戳的纯正文）兜底，`_assemble_transcript` **绝不回退到带标记的原文**；有降级块时进度里提示可单独「重新生成」。
- 分块 `CORRECT_MAX_CHARS` 1800 → 2800，降低并发块数、利于同一人发言合并。

### 2.2 纲目：回到 v1 细颗粒度 + 修正编号与时间
- **背景**：v0.5.0 改成"全程时间轴 digest 单次成稿"，结果**太概要**；而 v1 的"分段输入"颗粒度更细、更合用户预期（虽然 v1 阶段编号乱、时间不准）。
- **新做法**：回到 **v1 的分段生成**并补上修正：
  - 把**带时间戳的原始稿**按 `CHAPTERS_SEG_CHARS=6000` 分段，每段用 `chapters_segment_system` **细致划分**多个阶段（明确要求"宁可多分、聚焦小主题"），段内从「阶段1」起编号；
  - 各段拼接后 `_renumber_stages` **统一重排**为连续 `### 阶段1..K`；
  - 时间区间由模型从该段内的时间戳就地标注，**不臆造、无对应留空**；无时间戳时退用实录内容、时间留空。
- 既找回 v1 细分割，又修掉 v1 的编号乱、时间不准。

## 4. P5 智能问答（课堂答疑）—— 规划

**目标**：工作台提供独立的「提问」窗口，用户就本堂课自由发问，AI 依据**该会话的素材（原始转写稿 / 辅助资料）+ 已生成产出（实录、纲目、撷要、笺注、脉络）**作答，做到「有据可查、不杜撰」。

**排期决策**：安排在 **P3 辅助素材之后**。理由——问答要「根据提供的资料回答」，而当前后端只消化 txt/md，PDF/图片素材要等 P3 接入后才能真正进入问答上下文；产出侧已具备，可直接引用。P3 完成即解锁高质量问答的数据底座，P5 紧随其后。在 P3 之前若要先做，可先支持「仅基于已有产出 + txt/md 素材」的简化版。

**实现要点（拟）**：
- 后端：新增 `POST /api/sessions/{id}/ask`（body `{question, history?}`，SSE 流式返回）。上下文 = 选取相关产出 + 素材摘要拼装进 system/user；超长按需检索/截断（先简单拼接，后续可加轻量检索）。问答历史落盘 `data/sessions/<id>/qa/`，刷新可续。
- Provider：复用现有模型层与会话级模型覆盖（问答可单独选模型）；安全红线沿用「不杜撰数字/人名/公司名，答不到就说没有」。
- 前端：工作台新增「问答」Tab 或侧边抽屉，聊天式 UI，复用 `attachStream` 接 SSE；引用来源可标注「出自：笺注 / 纲目 阶段3」。
- 提示词：新增 `qa_system/qa_user`，强调「只依据给定资料与产出回答，超出范围明确说明」。

## 6. 已知小问题

- Markdown 渲染未消毒（`dangerouslySetInnerHTML`）——本地单用户风险低，上云/多用户阶段加 DOMPurify。
- 修订基于当前版本内容，不重新读原始素材；编辑型修订足够，若需"回到原文重写"待扩展。
- 删除「当前默认」供应商时默认回退到列表第一个，建议删前先改默认。
- 实录角色标注依赖模型语义判断：时间戳已确定性清除；但发言人标签在"模型成功但未遵守"时仍可能偶现（空返回/失败的块已由兜底彻底剥离）。若仍困扰，可加"确定性剥离发言人标签、纯靠语义判断"的开关（会略降角色识别准度）。
- 纲目分段在段边界可能把同一主题切成相邻两个阶段（细颗粒度的代价，符合 v1 风格）。
