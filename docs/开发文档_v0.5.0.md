## 2. v0.5.0 关键变化（相对 v0.4.0）

### 2.1 实录：区分发言人 · 不切断连续发言 · 去时间戳
- **诉求**：① 去掉时间戳标记；② **保留并区分发言人**——不同发言人之间明显区隔，但同一发言人一段时间内连续的发言要保持完整、不被切碎。
- **做法（核心是切分维度换成「说话人轮次」）**：
  - 新增 `transcript.parse_turns`：逐行解析为「说话人轮次」，同一发言人连续行合并为一轮，切换发言人才开新轮 → 从结构上保证连续发言不被打断。
  - 重写 `transcript.build_correction_units`：纠错单元**永不在一个轮次中间切断**；多个短轮次可打包进同一单元（各自带标签），仅当单个轮次超过 `max_chars` 才按句切分，后续片标 `is_continuation`。
  - 说话人识别 `_match_speaker` 兼容 `发言人1` / `发言人 1` / `主持人：` / `发言人2：00:01:10` / `主持人 00:02:00`（时间戳独占行）/ `Speaker 1:` / `A：` 等格式；为降低误判，关键词须伴随「编号/时间戳/冒号」之一（规避「老师讲得好」这类裸角色词），字母代号须带冒号。
  - `prompts.transcript_system` 改为：去时间戳但**保留说话人标签**，每位发言人以加粗 `**发言人X：**` 起段、不同发言人空行区隔、同一发言人连续发言并为一段；无标签片段不杜撰发言人。
  - `engine._assemble_transcript`：`is_continuation` 片段无缝接回上一段；标题统一 `# 实录`，不再有时间桶小标题。

### 2.2 纲目：阶段连续化 + 时间从原文对位
- **问题**：长稿时阶段出现「1–6 又从 1–5」碎片化（分段各自从「阶段1」重新计数后朴素拼接）；且实录已去时间戳，时间需另寻来源。
- **做法**：
  - 新增 `engine._make_chapters`：长稿走 **map-reduce**——先分段提炼「话题脉络条目（不编号）」，再**单次**汇编成贯穿全程的连续阶段，从根上消除重复计数；短稿仍单次生成。
  - 新增 `transcript.build_timeline_digest`：从**原始转写稿**逐行提取 `[HH:MM:SS] 片段开头` 时间锚点（时间戳在行首或说话人标签后都能识别，过多时均匀抽样），作为「时间轴参考」传入纲目。
  - `prompts.chapters_system/chapters_user` 改版：阶段编号 1 起连续贯穿全程、合并碎片；时间区间**只依据时间轴参考**对位标注，无参考则留空括号。

### 2.3 状态徽标随进度 + 单一版本源

- 前端徽标显示 `后端在线 · v0.5.0 · P2`（之前固定为 `P1`）。

## 4. P5 智能问答（课堂答疑）—— 规划

**目标**：工作台提供独立的「提问」窗口，用户就本堂课自由发问，AI 依据**该会话的素材（原始转写稿 / 辅助资料）+ 已生成产出（实录、纲目、撷要、笺注、脉络）**作答，做到「有据可查、不杜撰」。

**排期决策**：安排在 **P3 辅助素材之后**。理由——问答要「根据提供的资料回答」，而当前后端只消化 txt/md，PDF/图片素材要等 P3 接入后才能真正进入问答上下文；产出侧已具备，可直接引用。P3 完成即解锁高质量问答的数据底座，P5 紧随其后。在 P3 之前若要先做，可先支持「仅基于已有产出 + txt/md 素材」的简化版。

**实现要点（拟）**：
- 后端：新增 `POST /api/sessions/{id}/ask`（body `{question, history?}`，SSE 流式返回）。上下文 = 选取相关产出 + 素材摘要拼装进 system/user；超长按需检索/截断（先简单拼接，后续可加轻量检索）。问答历史落盘 `data/sessions/<id>/qa/`，刷新可续。
- Provider：复用现有模型层与会话级模型覆盖（问答可单独选模型）；安全红线沿用「不杜撰数字/人名/公司名，答不到就说没有」。
- 前端：工作台新增「问答」Tab 或侧边抽屉，聊天式 UI，复用 `attachStream` 接 SSE；引用来源可标注「出自：笺注 / 纲目 阶段3」。
- 提示词：新增 `qa_system/qa_user`，强调「只依据给定资料与产出回答，超出范围明确说明」。

## 6. 已知小问题

- Markdown 渲染未消毒（`dangerouslySetInnerHTML`）——本地单用户风险低，上云/多用户阶段加 DOMPurify。
- 修订基于当前版本内容，不重新读原始素材；编辑型修订足够，若需"回到原文重写"待扩展。
- 删除「当前默认」供应商时默认回退到列表第一个，建议删前先改默认。
- 说话人识别基于常见格式启发式；非常规标注（如纯姓名无冒号、缩写无规律）可能识别不全，会退化为不分段的连贯正文。
