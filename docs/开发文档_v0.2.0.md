## 2. v0.2.0 关键变化（相对 v0.1.0）

### 2.1 五产出取代四产出，文艺命名
| 旧（v0.1.x） | 新（v0.2.0） | 说明 |
|---|---|---|
| 纠错逐字稿 | **实录** | 基于原始转写稿去口水/纠错/分段 |
| 章节稿 | **纲目** | 时间顺序梳理章节 |
| 纪要主体 | **撷要** + **笺注** | 拆成精炼版与详尽版两个独立产出 |
| 知识图谱 | **脉络** | 横版 Mermaid flowchart LR |

### 2.2 每个产出独立生成、独立选模型
- API：`POST /api/sessions/{id}/run-step?step=<key>`（幂等 · 同步触发）
- API：`GET /api/sessions/{id}/run-step-stream?step=<key>`（SSE 单步进度）
- API：`PUT /api/sessions/{id}/step-model`（设置某步使用的 provider/model；空 = 重置默认）
- 后台 Runner（`pipeline/runner.py`）：`(sid, step_key)` 维度的线程任务，事件落盘到 `data/sessions/<sid>/progress/<step>.json`，刷新页面/换浏览器都能恢复进度。
- 依赖：`StepDef.requires` / `requires_any` 引擎前置校验，未满足直接抛事件。

### 2.3 横版 Tab + 加宽 + 老产出兼容
- 工作台铺满浏览器宽度（去掉 max-width: 920px）。
- 5 个 Tab 横向铺开 → 点击后内容在下方展开（之前是左 Tab/右面板的纵向模式）。
- 老会话存的「逐字稿/章节稿/纪要主体/知识图谱」自动映射回新名（`engine.py::LEGACY_NAMES`），`available_artifacts(sid)` 实时扫盘，`meta.artifacts` 不再权威。

### 2.4 默认 DeepSeek + Karl-5 模型清单收敛
- 全局默认 provider 改为 `deepseek`，default_model = `deepseek-v4-flash`。
- DeepSeek 模型列表更新为 `deepseek-v4-flash` / `deepseek-v4-pro`（旧 `deepseek-chat/reasoner` 已于 2026-07-24 弃用）。
- `anthropic_compat`（Karl-5 中转站）仅保留 `claude-opus-4-8` / `claude-sonnet-4-6`，去掉 Haiku。
- 中转站经 Cloudflare 被 403 → Provider 自动注入浏览器 UA 头绕过 WAF。

### 2.5 脉络图改造
- Mermaid 主题：暖米白底 + 珊瑚边/连线 + 衬线字体；`flowchart LR`（横版）；A4 长宽比画布；4 角珊瑚渐晕。
- **矢量缩放永远清晰**：关键开关 `htmlLabels: false` —— 节点文字走原生 SVG `<text>`，避免 `<foreignObject>` 内的 HTML 在 transform: scale 下被光栅化。
- 滚轮缩放以鼠标位置为锚点；＋／－ 按钮以画布中心缩放；缩放上限 600%。
- 滚轮事件用 `addEventListener("wheel", fn, { passive: false })` + `overscroll-behavior: contain` + `touch-action: none`，画布内滚轮不再带动浏览器页面。

### 2.6 输入素材区改造
- 自定义上传按钮（虚线珊瑚边 + ＋图标）替代原生 `<input type="file">` 默认样式。
- 文件支持删除、重命名（保留扩展名）：
  - `DELETE /api/sessions/{id}/inputs/{filename}`
  - `PUT /api/sessions/{id}/inputs/{filename}` body `{new_name}`

## 5. 关键设计沿用 / 强化

- 本地优先、前后端分离、存储层接口化（上云时换实现）。
- 模型 Provider 可切换、每步独立选模型；运行时未配置 provider 时报「请检查 .env 或在该产出处切换模型」而非失败崩溃。
- 板块化：新增板块只写 `config.py + prompts.py`，引擎复用。
- **产出文件名稳定，命名变化用 `LEGACY_NAMES` 维护历史会话兼容**（这是首次升级未来产出名时的样板）。
- 后台任务事件全部落盘 → 浏览器关闭/网络中断/服务重启都能 resume。
- raw 输入与产出**永不入 git、永不外发**：`.gitignore` 包含 `data/sessions/`、`backend/.env`。

## 6. 下一步：P2 计划

1. **持续迭代修订**：每个产出（实录/纲目/撷要/笺注/脉络）下方常驻修订窗口，AI 按用户意见对该产出再生成新版本，保留版本历史与对应意见。
2. 修订时仍走单步 SSE，复用现有 runner / progress 通道。
3. 路由：`POST /api/sessions/{id}/revise-step?step=X` body `{instruction}`。
4. 前端：版本切换器（v1 / v2 / v3 …）+ 修订意见输入框。
5. P2 完成后升版 `开发文档_v0.3.0.md`。
