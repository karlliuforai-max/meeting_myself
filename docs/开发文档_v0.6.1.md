## 2. 本版排序决策

用户当前测试数据主要是「转写文本 + 图片版课堂笔记」，暂时没有 PDF。基于这个现实，先做导出比先做 PDF 更能形成日常使用闭环：

- 已生成内容可以立即保存、归档、分享或放进知识库。
- PDF 抽文本没有真实样例时不易验收抽取质量，顺延更稳。
- 当前导出先保持轻量，不提前卷入 DOCX/PDF 排版复杂度。

## 3. 导出能力设计

### 3.1 单个产出下载

- API：`GET /api/sessions/{sid}/artifacts/{name}/download`
- 下载当前产出文件，文件名沿用 `实录.md`、`纲目.md`、`撷要.md`、`笺注.md`、`脉络.mmd`。
- 兼容老会话：若规范新文件名不存在，会按 `LEGACY_NAMES` 查找旧文件名内容。
- 响应头使用 `filename*` 支持中文文件名，同时提供 ASCII fallback。

### 3.2 历史版本下载

- API：`GET /api/sessions/{sid}/artifacts/{name}/versions/{version}/download`
- 前端查看历史版本时，按钮显示「下载此版本」。
- 文件名自动追加版本号，例如 `撷要_v1.md`。

### 3.3 项目产出 ZIP 打包

- API：`GET /api/sessions/{sid}/exports/bundle`
- ZIP 内容：当前项目所有已生成产出。
- 不包含原始输入素材（转写稿、图片、未来 PDF），避免把测试数据无意带出。
- ZIP 文件名为 `{项目名}_产出.zip`，非法路径字符会替换为 `_`。
