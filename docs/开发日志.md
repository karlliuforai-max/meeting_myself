### 第一轮实现（中间态）
- 五产出 + 文艺命名：实录 / 纲目 / 撷要 / 笺注 / 脉络。
- 撷要（精炼）和笺注（详尽）拆成两个独立产出。
- 单步生成接口 `run-step` + SSE `run-step-stream` + 后台 `runner.py`（事件落盘到 `progress/<step>.json`）。
- 老会话兼容：`LEGACY_NAMES` 把老文件名映射回新名。
- DeepSeek 默认 + 模型名更新；Karl-5 收敛为两款；中转站 UA 注入修 403。
- 横版 Mermaid `flowchart LR`、A4 长宽比、暖色主题、可缩放可拖拽视图。

### 用户复测后又指出 5 个问题
1. 老 session 的产出系统不识别 → 改 `_session_public` 实时调 `available_artifacts(sid)` 扫盘，不再依赖 `meta.artifacts` 落后字段。
2. Tab 副标默认要展示 DeepSeek + v4-flash，而不是「未选模型」→ 加 `defaultProvider/defaultModelName` 回落。
3. Tab 想转置成横版（顶部 Tab 行 + 下方面板）→ `flex-direction: column`，标签横向 wrap。
4. 界面只用 2/3 宽 → 去掉 `.content` `max-width: 920px`。
5. 脉络放大 200% 文字模糊 → 关键开关 `htmlLabels: false`，节点改用原生 SVG `<text>`，矢量缩放永远清晰；同时 SVG 加 `preserveAspectRatio="xMidYMid meet"`，stage 的 padding 移到外层 canvas。

### 又一轮 3 个小问题
1. 上传按钮难看 → 自定义虚线珊瑚边按钮 + 隐藏原生 input。
2. 文件支持删除/重命名 → 后端加 `DELETE` 与 `PUT /inputs/{filename}`，前端文件列表每行带改名/删除按钮。
3. 滚轮缩放时浏览器页面也在滚 → React 合成 wheel 事件默认 passive，`preventDefault` 不生效；改用 `addEventListener("wheel", fn, {passive: false})` + `overscroll-behavior: contain` + `touch-action: none` 三层防护。

## 2026-06-15 · v0.4.0：P2 持续迭代修订

### 起点：审视进度，定下一步
- 用户让我整体审视开发进度，并重新审视 P0-P4 是否需调整。

### 方案取舍（修订）
