# 任务背景
我高频地会有一些交流场景，例如我正在北大光华商学院上学，会有许多课堂讲座；我作为一名矿业公司的CEO，会有一些商务交流的场景；我作为一名知识Vlog博主，会有一些跟身边朋友闲散交流，但其中内涵价值很高的聊天等。我未来会把这些录音录制下来，转写成文稿。现在我希望将他们通通整理成一些对我量身定制的，可用的会议纪要，供我未来备忘和使用
# 项目设计
## 呈现形式
我希望暂时做成一个以网页形式呈现的会议纪要生成平台；
## 网页结构
- 由于我的应用场景有所不同，对应的语种、会议纪要模板和输出要求都不同，因此我希望在一个平台中，区分多个板块，用以操作不同的场景；
- 目前常用的3个场景即：普通话/英语为主的商学院课堂和讲座；四川话/乐山话偶尔夹杂普通话的矿业商务沟通；日常聊天。我希望平台先支持这三个板块的操作，并且预留“通用”场景，用以输出通用会议纪要；预留扩展开发端口，用于未来其他场景的开发集成。
## 开发规划
功能上，我希望一个板块一个板块进行开发，优先开发商学院课堂讲座这一个板块（你可以取一个更好听一点的名字），其余板块在平台上留位置，但不着急开发

# 针对商学院课堂讲座功能板块的开发
## 输入
- 输入的内容：用录音笔录音后，由AI转写好的转写文字稿（由于录音和转写效果，可能会有一些错误）；我自己的课堂笔记；其他的辅助资料
- 输入的格式：主要包含txt（转写文字稿）、pdf（主要是辅助资料）和图片（主要是课堂手写笔记）。后续工作中会以文字稿为主体进行处理，辅助资料和课堂笔记需要理解后辅助输出件更加高质量输出
## 输出
1.  纠错并整理后的逐字稿。由于原版AI转写的文字稿存在两个问题：
    1. 由于录音和转写的不准确，导致的词句识别错误，如人名、公司名、专有名词、技术名词、常规近音词等。需要通过上下文语义判断并纠错
    2. 由于录音文件及转写文稿本身带时间戳，同时有许多无意义的嗯，啊等口语化和会议节奏临时中断带来的杂音和无意义词的录入，所以会把内容切割地很杂乱。因此需要整理其结构：
        1. 合并原有按秒级的时间戳，改为10分钟一级的时间戳
        2. 整理其内容，在尽量不改变原话的基础上，将一些结合上下文实在无意义和杂乱的嗯，啊等文字去掉，让逐字稿看起来更书面易读和流畅一些
2. 章节稿。由于整体会议时间很长，为了展现老师课堂和讲座的整体演讲逻辑。我需要按时间顺序对会议内容的整个流程进行总结。形式例如：
    第一阶段主题（xx：xx-xx:xx）
    核心内容总结
    第二阶段主题（xx：xx-xx:xx）
    核心内容总结
    第三阶段主题（xx：xx-xx:xx）
    核心内容总结
    ...
3.  课堂讲座纪要主体。需要包含核心内容和知识点及知识体系的总结。这个具体是种什么形式我没想好，你可以跟我一起讨论一下方案
4. 知识图谱（逻辑图）。即以逻辑图谱的形式，呈现整堂课老师的观点和论证过程，这其中包含了老师的核心观点（可能一堂课有多个），支撑这些个核心观点的子论点，以及支撑各个论点的论据。结果要体现他们分别是一种怎样的链接关系。这个部分不建议直接用AI生图的方式输出，建议用一种能够用代码画流程图的方式输出，我们也可以讨论下具体方案。

这就是我目前大概的一些想法，请你跟我先针对这个想法进行充分讨论，确定方案后形成开发文档再行开发。另外请你为这个项目取一个酷一点的名字！
