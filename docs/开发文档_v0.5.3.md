## 2. v0.5.3 关键变化（相对 v0.5.2）

### 2.1 速度
- **撷要 / 笺注：整篇一次成稿**（`engine._make_minutes`）。去掉 map-reduce，把完整实录(+纲目)一次喂给模型——依赖现代模型长上下文（deepseek-v4-flash / claude-sonnet-4-6 均 1M 上下文）。长稿从多次调用降到 1 次：更快、且不丢跨段关联。移除 `SUMMARY_SINGLE_LIMIT`/`SUMMARY_PART_CHARS`。
- **纲目：分段并行**（`engine._make_chapters` + `_parallel_map`）。各段并发生成（上限 `MAX_PARALLEL=6`），不再串行；编号仍由 `_renumber_stages` 统一重排为连续。
- **实录：分块 2800 → 10000 字**（`CORRECT_MAX_CHARS`）。6.3 万字样例块数 16 → 7，调用更少、跨块割裂更少、对高并发返空的中转更友好。单块输出上限 `CORRECT_MAX_TOKENS` 提到 16000（v4-flash 支持 384K 输出，~1.8 倍余量防截断）。

### 2.2 质量
- 撷要/笺注看到全篇 → 跨段综合更好、无"要点的要点"信息损耗。
- 笺注输出上限 4096 → 8000 token，详尽版不易截断。
- 纲目保留 v1 细颗粒度（仍分段），不受影响。

### 2.3 分步默认模型（保留用户手动切换权）
- 默认搭配：实录 / 纲目 → **deepseek-v4-flash**（量大、偏体力活，用快模型）；撷要 / 笺注 / 脉络 → **claude-sonnet-4-6**（偏归纳推理，用强模型）。
- 实现：`StepDef` 新增 `default_model` 字段；`engine._step_default` 按该"首选模型名"在【已配置(有 key)】的供应商里自动匹配（全局默认 provider 优先），匹配不到回退全局默认——**不写死供应商 id**，兼容用户自定义供应商。前端 `Workbench.stepDefault` 同逻辑展示默认值。
- **完整保留**每个产出处手动选 provider/model 的下拉；仅"未手动选"时的默认值变智能。

### 2.4 取舍
- 撷要/笺注**不再有 map-reduce 兜底**：若手动选了小上下文模型 + 超长稿，由 provider 直接报"上下文超限"（改用大上下文模型即可），不静默降级。
- 实录/纲目仍分块/分段：实录是等长改写、受速度与可靠性约束（非输入上限）；纲目分段是为细颗粒度。

## 3. 产出依赖与默认模型一览（v0.5.3）

| 产出 | 执行前置 | 内容素材 | 生成方式 | 默认模型 |
|------|---------|---------|---------|---------|
| 实录 transcript | 无 | 转写原文 | 10000 字分块 · 并行纠错 · 空返回重试+兜底 | deepseek-v4-flash |
| 纲目 chapters | 无 | 转写原文（带时间戳）/ 实录 | 6000 字分段 · 并行 · 编号重排 | deepseek-v4-flash |
| 撷要 minutes_concise | 实录 | 实录(+纲目) | 整篇一次成稿 | claude-sonnet-4-6 |
| 笺注 minutes_detailed | 实录 | 实录(+纲目) | 整篇一次成稿（max 8000 token） | claude-sonnet-4-6 |
| 脉络 graph | 撷要或笺注 | 纪要正文 | 单次 | claude-sonnet-4-6 |

> 可并行：实录 ∥ 纲目。默认模型可在每个产出处手动覆盖。

## 4. P5 智能问答（课堂答疑）—— 规划

**目标**：工作台提供独立的「提问」窗口，用户就本堂课自由发问，AI 依据**素材 + 已生成产出**作答，做到「有据可查、不杜撰」。

**排期**：安排在 **P3 辅助素材之后**（问答要"据资料回答"，PDF/图片素材需先由 P3 接入；产出侧已具备）。

**实现要点（拟）**：`POST /api/sessions/{id}/ask`（SSE）；上下文 = 相关产出 + 素材摘要；问答历史落盘 `data/sessions/<id>/qa/`；前端聊天式 Tab、复用 `attachStream`；新增 `qa_system/qa_user` 提示词强调"只依据给定资料回答"。

## 5. 下一步候选

1. **P3 辅助素材**（建议优先）：PDF 用 `pypdf` 抽文本、图片走视觉模型（Provider 已有 `supports_vision`），注入各产出提示词。也是 P5 问答的数据底座。
2. **导出**：md / docx / pdf，脉络图 PNG/SVG，整场打包 zip。
3. **P5 智能问答**：见 §4，排在 P3 之后。
4. **生成用时记录**（轻量）：记录并展示每次产出耗时。后端 `done` 事件附 `elapsed` + 所用 provider/model（可入版本 note）；前端面板/Tab 显示「本次生成 12.3s」，历史版本亦可见。进阶可记"墙钟 vs 累计 token"。
5. **实录分块截断保护**（健壮性，配合大块）：用户手动换成小输出上限模型时，等长改写会静默截断。计划：`finish_reason==length`（或输出贴近 `max_tokens`）时自动二分重切重试；超长稿（块数 > 并发）适度提高 `MAX_PARALLEL` 而非缩小块。
6. **P4**：术语库自进化、其余板块、设置页。

（实录/纲目内容机制见 [v0.5.1](开发文档_v0.5.1.md)；依赖解耦见 [v0.5.2](开发文档_v0.5.2.md)；P2 修订见 [v0.4.0](开发文档_v0.4.0.md)。）

## 6. 已知小问题

- Markdown 渲染未消毒（`dangerouslySetInnerHTML`）——本地单用户风险低，上云/多用户阶段加 DOMPurify。
- 实录角色标注依赖模型语义判断；时间戳已确定性清除，发言人标签在"模型成功但未遵守"时偶现（空返回/失败块已由兜底剥离）。
- 实录大块（10000 字）配 16000 token 输出对默认模型安全；手动换小输出上限模型有静默截断风险（见 §5.5 计划）。
- 纲目分段在段边界可能把同一主题切成相邻两个阶段（细颗粒度代价，符合 v1 风格）。
- 删除「当前默认」供应商时默认回退到列表第一个，建议删前先改默认。
