## 2. v0.3.0 关键变化（相对 v0.2.0）

### 2.1 模型配置中心：Provider 从 .env 静态 → 运行时 store 持久化
这是本版最大的架构调整。此前四个 Provider 写死在 registry、配置只读 `.env`；现在改为运行时可增删改并持久化。

- **持久化存储** `providers/store.py`：配置落盘 `data/providers.json`，结构 `{default_id, providers:[...]}`。每条 config：`{id, label, kind(openai|anthropic), base_url, api_key, models[], default_model, supports_vision}`。
- **首次播种**：`providers.json` 不存在时，从既有 `.env` 播种四个供应商，**沿用稳定 id**（`claude/anthropic_compat/deepseek/openai_compat`），保证历史会话 `step_models` 引用仍有效。之后改 `.env` 不再生效（除非删 `providers.json` 重新播种）。
- **动态构建** `providers/dynamic.py`：按 config 实时构建 Provider，按 `kind` 复用既有两套 chat 调用逻辑——`_AnthropicBase`（Anthropic Messages）/ `_OpenAICompatBase`（OpenAI Chat Completions）。中转站（有自定义 base_url 的 anthropic）自动注入浏览器 UA 绕过 WAF。
- **registry 重写**：从 store 实时构建；`list_providers()` **不外泄 api_key**（只给 `has_key` 标记），编辑时经 `GET /providers/{id}` 详情回填。
- **前端面板** `components/ModelConfig.jsx`：右上角 ⚙ 打开两栏弹窗（左供应商列表 / 右编辑表单）。支持新增、编辑、删除、设默认、连通测试（可测**未保存的草稿**）。API Key 明文/掩码切换、默认模型下拉、视觉开关。

### 2.2 项目与素材原地重命名
- 通用组件 `components/InlineEdit.jsx`：挂载即聚焦全选，回车/失焦保存、Esc 取消。
- 三处接入：项目列表项、工作台标题、上传素材文件名（改名保留扩展名）。双击名称亦可进入编辑。
- 后端复用既有 `PATCH /sessions/{id}`（title）与 `PUT /sessions/{id}/inputs/{filename}`（new_name）。

### 2.3 审查修复（健壮性 / 一致性）
- **会话元数据并发安全**：`SessionStore` 加 `RLock`；`_write_meta` 原子写（临时文件 + `os.replace`）；新增 `mutate(sid, fn)` 在锁内读改写。改名、step-model 路由改走 `mutate`，杜绝多产出并行结束时互相覆盖 `meta.json`。
- **Runner 内存回收**：`(sid, step)` 任务完成即从 `_RUNS` 移除（事件已落盘，迟到订阅经 `subscribe` 回退 `load_history` 读盘）。
- **死代码清理**：删除四个不再使用的具体 Provider 类，`claude.py` / `openai_compat.py` 仅保留被复用的调用基类。
- **失效字段清理**：移除 `StepDef.default_provider/default_model`（默认由 store 默认项决定）。
- **文案**：生成失败由「检查 .env」改为「在右上角模型配置面板补全」。
- **误入文件**：删 `backend/package-lock.json`、空目录 `backend/revision/`。
- **文档**：README / `.env.example` 注明 `.env` 仅首次播种。

### 2.4 配置面板 UI 细节
- 接口协议命名专业化：`OpenAI Chat Completions` / `Anthropic Messages`。
- API Key 眼睛图标明文/掩码切换。
- 默认模型由 `datalist`（点不动）改为真正可点选的 `<select>`。
- 修复「支持视觉」复选框被全局 `input{width:100%;margin-top}` 撑坏的换行排版（加 `input[type=checkbox]` 重置 + 整行做成带边框设置项）。

## 5. 关键设计沿用 / 强化

- 本地优先、前后端分离、存储层接口化（上云时换实现）。
- **Provider 配置运行时化**：增删改即时生效、持久化、密钥不外泄；老会话凭稳定 id 兼容。
- 每步独立选模型；默认模型由「模型配置面板」的 store 默认项统一决定。
- 板块化：新增板块只写 `config.py + prompts.py`，引擎复用。
- 产出文件名稳定，命名变化用 `LEGACY_NAMES` 维护历史会话兼容。
- 后台任务事件全部落盘 → 浏览器关闭/网络中断/服务重启都能 resume。
- raw 输入、产出、`providers.json`（含密钥）**永不入 git、永不外发**：`.gitignore` 覆盖 `data/`、`**/.env`。

## 7. 已知小问题（不阻塞 v0.3.0）

- **Markdown 渲染未消毒**：`OutputView` 用 `dangerouslySetInnerHTML` 直出 marked 结果。本地单用户风险低；上云/多用户阶段加 DOMPurify。
- `.env` 旧值含尾随 `#` 注释不会被剥离——但 `.env` 现仅首次播种用，影响面已收窄；面板填写不受影响。
- 删除「当前默认」供应商时，默认自动回退到列表第一个，建议删前先改默认。
