# 会议纪要生成平台 · 开发文档 v1

> **版本说明**：本项目开发文档按版本号管理。每推进到一个阶段性里程碑，新建
> `开发文档_v{n}.md` 重写当前开发状态，旧版本保留不覆盖。本文件 v1 = 方案确认稿（开发起点）。
> 配套背景见 [任务背景.md](任务背景.md)，沟通时间线见 [开发日志.md](开发日志.md)。

**v1 状态**：方案已确认；前端路线 A（React+Vite）已确认；准备进入 P0 脚手架。

## 1. 项目目标

把不同场景的录音转写稿，整理成"量身定制、可直接备忘使用"的会议纪要。
平台按**场景板块**划分，先做**商学院课堂讲座**板块，其余板块在平台上预留入口、暂不开发。

### 板块规划

| 板块 | 语种 | 状态 |
|------|------|------|
| 商学院课堂讲座（暂名"学堂"） | 普通话 / 英语 | **本期开发** |
| 矿业商务沟通 | 四川话/乐山话+普通话 | 预留入口（已有 `sichuan-mining-transcribe` 工具，未来接入） |
| 日常聊天 | 普通话 | 预留入口 |
| 通用 | 任意 | 预留入口 |

> 设计原则：板块 = 一套「语种配置 + 提示词模板 + 输出结构 + 默认模型」。
> 平台核心（处理引擎、模型层、修订机制、存储、网页外壳）所有板块复用，新增板块只写配置与模板。

## 2. 总体架构

```
┌─────────────────────────────────────────────┐
│  前端 SPA（浏览器）                            │
│  板块选择 · 上传 · 前置提示 · 进度 · 四产出标签页 · 修订窗口 · 导出 │
└───────────────┬─────────────────────────────┘
                │ HTTP / SSE(进度流)
┌───────────────▼─────────────────────────────┐
│  后端 FastAPI（本地运行）                       │
│  ┌──────────┐  ┌──────────────┐  ┌──────────┐ │
│  │ 板块注册表 │  │  处理流水线    │  │ 会话存储  │ │
│  │ modules/  │  │  pipeline/    │  │ data/    │ │
│  └──────────┘  └──────┬───────┘  └──────────┘ │
│              ┌─────────▼─────────┐             │
│              │ 模型 Provider 抽象层 │            │
│              │ Claude/DeepSeek/兼容 │            │
│              └───────────────────┘             │
└─────────────────────────────────────────────┘
```

- **运行形态**：先本地（后端跑在本机，浏览器访问 `localhost`），前后端分离，预留上云。敏感录音/转写稿不出本机。
- **上云预留**：存储层与鉴权做成可替换接口（本地文件 → 未来对象存储 + 登录）。

## 3. 技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 后端 | Python 3.10+ / FastAPI / uvicorn | 与现有 `sichuan-mining-transcribe` 一致，便于复用引擎与术语库思路；async 适合进度流式 |
| 模型 | Anthropic SDK + OpenAI 兼容 SDK | 默认 Claude；可切换 DeepSeek 等 |
| 前端 | Vite + React + Tailwind | 标签页/进度/修订交互较多，SPA 更顺手 |
| 渲染 | react-markdown（纪要/逐字稿）、mermaid（知识图谱） | Markdown + 代码画图，无需 AI 生图 |
| 文档解析 | pypdf（PDF）、模型视觉能力（手写笔记图片） | 与现有工具一致 |
| 存储 | 本地文件系统（Markdown + JSON） | 本地优先、易查看、与现有工具一致 |

## 4. 模型 Provider 抽象层（关键）

需求：**用户可自由切换模型，默认 Claude**；且支持视觉（看懂 PDF/手写笔记图片）。

```
providers/
  base.py          # 统一接口：chat(messages, model, **opts) / supports_vision
  claude.py        # Anthropic SDK，默认
  deepseek.py      # OpenAI 兼容
  openai_compat.py # 任意 OpenAI 兼容端点（通用兜底）
  registry.py      # 按名取 provider；从 .env / 设置页读 key
```

- **每个处理步骤可独立配置模型**（例：逐字稿纠错用便宜模型，纪要主体/图谱用 Claude）。默认全用 Claude。
- 配置来源：`.env`（API Key）+ 前端「设置」页（选模型、切 provider）。
- 视觉任务（手写笔记、PDF 扫描件）自动要求 `supports_vision` 的 provider。

## 5. 商学院板块 · 处理流水线

```
输入：txt转写稿(主体) + pdf辅料 + 图片课堂笔记
  │
  ├─[前置提示词] 用户补充背景 & 重点要求 —— 注入下列每一步
  │
  ├─ Step0 解析：拆时间戳/分块；PDF抽文本；图片走视觉模型转文字
  ├─ Step1 纠错+整理 ──▶ 产出① 逐字稿（10分钟级时间戳、去口水词、纠正音近错字/人名/术语）
  ├─ Step2 时序切分 ──▶ 产出② 章节稿（阶段主题 + 时间区间 + 核心总结）
  ├─ Step3 知识综合 ──▶ 产出③ 纪要主体（见 §6）
  └─ Step4 论证抽取 ──▶ 产出④ 知识图谱（Mermaid 代码图，见 §7）
  │
  └─[修订循环] 每个产出可常驻反复修改，直到满意（见 §8）
```

处理过程**前台流式展示进度**（SSE），参考现有工具"每 5 秒打印进度"的体验。
分块并行 + 断点续传：长会议（数万字）分段并行处理，中断可续跑。

## 6. 产出③ 纪要主体 · 结构（已确认）

1. **一句话概要** —— 本堂课最重要的结论
2. **核心知识点 / 框架模型** —— 逐条：概念名 → 老师阐释 → 适用场景/边界
3. **关键案例与数据** —— 公司案例、关键数字、研究结论
4. **金句 / 重要论断**
5. **对你的启发与行动项** —— 结合 CEO/商学院身份的个性化提炼（**独立成节**）
6. **延伸思考与疑问**
7. **重点术语表（中英对照）** —— 商学院英文术语多，便于回看

> **篇幅可选**：生成时可选「精炼版」（快速备忘）或「详尽版」（接近完整复盘）。

## 7. 产出④ 知识图谱 · 方案（已确认）

- 用 **Mermaid** 代码画图，网页直接渲染、可版本管理、不用 AI 生图。
- 结构：核心观点（可多个）→ 子论点 → 论据；**边上标注链接关系**（支撑 / 递进 / 因果 / 对比）。
- 同时保存 `.mmd` 源码，便于导出与二次编辑。

## 8. 两个协同功能（本期重点）

### 8.1 任务前 · 自定义补充提示词
- 每个会议处理前，提供「补充背景」+「重点要求」输入框。
- 内容注入**所有步骤**的提示词（背景信息 + "必须体现的重点"）。
- 随会话保存，重跑沿用。

### 8.2 任务后 · 持续迭代修订
- 四个产出**各自**一个**常驻**修订窗口（不一次性、不消失）。
- 一轮修订 = AI 接收【原始素材 + 前置提示词 + 当前版本 + 本次意见】→ 重新优化该产出。
- 每轮存为**新版本**，保留历史、可回退、可对比。
- 反复修订直至用户满意。

## 9. 数据与存储

```
data/sessions/<会话ID>/
  meta.json              # 板块、标题、创建时间、前置提示词、各步模型配置
  inputs/                # 原始 txt / pdf / 图片
  artifacts/
    逐字稿.md  章节稿.md  纪要主体.md  知识图谱.mmd
  versions/<产出名>/      # 每轮修订版本 + 对应的修订意见
  progress.json          # 断点续传状态
```

- 纯本地、Markdown 可直接查看；与现有工具风格一致。
- 上云时只替换存储后端接口。

## 11. 分阶段开发计划

| 阶段 | 内容 | 产出 |
|------|------|------|
| **P0 脚手架** | 后端骨架 + 模型 Provider 抽象层 + 板块注册表 + 前端壳 + 本地存储 | 能跑起来、能切模型、能建会话 |
| **P1 核心流水线** | 商学院板块：txt → 逐字稿 → 章节稿 → 纪要主体 → 知识图谱；进度流式 | 四产出可一键生成 |
| **P2 协同功能** | 前置自定义提示词 + 持续迭代修订（版本管理） | 人机协同打磨闭环 |
| **P3 辅助素材** | PDF 辅料 + 图片手写笔记（视觉模型）接入流水线 | 多源输入增强质量 |
| **P4 完善** | 导出（md/pdf）、术语库自进化、其余板块占位入口、设置页 | 可日常使用 |

## 12. 安全红线（沿用现有工具约定）

- 不改动数字、金额、日期、人名、公司名；涉数字的模型变更进「疑点清单」。
- 原始录音/转写稿与 `data/` 不提交 Git、不外发。
