## 2. 需求与定位

用户高频场景为「**转写文字稿 + 课堂笔记照片**」。照片是辅助素材（手写笔记、板书、PPT、讲义），价值在于补全讲授中一带而过、但被记进笔记的要点/公式/数据。

**关键定位决策**：
- 照片 = **辅助素材**，转录文字注入做知识归纳的产出（撷要 / 笺注）。
- **不混入实录**：实录是讲话逐字稿，照片不是讲话，混入会污染逐字稿。
- **暂不进纲目**：纲目是按原文时间戳对位的时间轴骨架，照片无时间戳，强行注入会破坏时间对位；如确有需要，后续可作为"补充参考"接入汇编步。

## 3. 设计

### 3.1 图片 → 文字（视觉转录）

```
图片字节 ──Pillow 降采样(≤1568px)──► base64 ──► 视觉模型 chat(多模态) ──► Markdown 转录
```

- 转录提示词（`prompts.note_transcribe_*`）：完整还原文字与层级；公式转 LaTeX、表格转 Markdown；图示用文字描述结构；字迹不清标 `[字迹不清]`、不杜撰；数字/人名/公司名/英文术语照原样、拿不准标 `[存疑]`。
- Pillow 为可选依赖：缺失或处理失败时原图 base64 兜底。

### 3.2 Provider 多模态（向后兼容）

- `Message` 新增可选 `images: List[ImagePart]`（默认空）。无图时两套 `chat()` 仍发纯文本字符串 content，**既有行为不变**。
- 带图时：Anthropic 拼 `image` 内容块（base64 source）；OpenAI 兼容拼 `image_url`（`data:<mt>;base64,<data>`）。

### 3.3 按文件缓存

- 转录结果缓存到 `data/sessions/<id>/derived/notes/<file>.<mtime>.md`。
- 命中条件：图片仍在且 mtime 未变。重新上传/替换（mtime 变）→ 自动失效重转。
- 写缓存时清掉同名旧 mtime 文件，避免堆积。

### 3.4 图片识别模型选择（用户可手动指定）

- 模型配置面板底部「图片识别模型」选择器，全局生效、与各产出模型独立。
- 解析顺序（`vision.resolve_vision_provider`）：
  1. 用户手动指定且该供应商已配置 → 用它（尊重用户选择，不强制 `supports_vision`）；
  2. 否则自动选第一个【支持视觉且已配置】的供应商（全局默认优先）；
  3. 都没有 → 跳过图片，进度里提示去配置。
- 持久化在 `providers.json` 顶层 `vision_id` / `vision_model`（老文件缺字段 → 视为"自动"）。

| 产出 | 内容素材 | 笔记照片 | 默认模型 |

| 脉络 | 撷要或笺注 | （间接） | claude-sonnet-4-6 |

## 7. 已知取舍

- 笔记是辅助：与逐字稿冲突以逐字稿为准，提示词明确禁止据笔记杜撰。
- 转录质量受所选视觉模型影响；识别失败的单张会跳过并在进度提示。
- 动图/多帧仅取首帧；超大图降采样可能损失极细笔迹。
