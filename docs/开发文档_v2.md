## 3. P0 已实现的 API（已自测）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/health` | 健康检查 |
| GET | `/api/modules` | 板块列表（含 enabled 标志） |
| GET | `/api/providers` | 模型 Provider 列表 + 可用性 + 模型清单 |
| POST | `/api/providers/test` | 切换模型时的连通性测试（最小调用） |
| POST | `/api/sessions` | 新建会话（占位板块会被拒绝） |
| GET/PATCH/DELETE | `/api/sessions/{id}` | 查/改(标题·前置提示词)/删 |
| GET | `/api/sessions` | 会话列表（可按板块过滤） |
| POST | `/api/sessions/{id}/inputs` | 上传输入文件（多文件） |
| GET | `/api/sessions/{id}/artifacts/{name}` | 读产出 + 版本列表 |
| GET | `.../artifacts/{name}/versions/{v}` | 读指定历史版本 |

## 5. 关键设计沿用（不变，详见 v1）

- 本地优先、前后端分离、预留上云；存储层接口化便于替换。
- 模型 Provider 可切换、默认 Claude、每步可独立选模型、视觉任务用支持视觉的 provider。
- 板块化：新增板块只写 config + prompts，核心引擎复用。
- 纪要主体 7 节（含中英术语表）、精炼/详尽可选；知识图谱用 Mermaid。
- 两大协同功能：前置提示词（数据通道已建：`SessionMeta.pre_prompt`）+ 持续迭代修订（版本存储已建：`storage` 的 versions）。
